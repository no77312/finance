# 产品设计：持仓圈

## 产品定位

持仓圈是一个小群体持仓同步工具，适合朋友、研究小组、家庭资产讨论小组或投资社群内部使用。它解决的问题不是“荐股”，而是让成员在一个相对可信的空间里同步自己真实或半公开的仓位、理由和风险。

## 首版目标

- 成员可以创建或进入一个群组。
- 成员可以提交自己的持仓，包括标的、市场、数量、成本、现价、币种、备注。
- 群组成员可以查看大家提交的持仓。
- 群组可以看到按币种汇总的市值和浮盈亏。
- 群组可以看到“共识标的”：同一个标的有多少人持有、合计市值和浮盈亏比例。
- 每个成员可以管理自己提交的持仓。
- 用户可以控制单笔持仓的可见性。

## MVP 用户流程

1. 打开 App，进入默认群组“长期主义小组”。
2. 在“总览”查看资产概览、共识标的、最新提交。
3. 点击右上角加号提交持仓。
4. 切到“我的”查看和编辑自己的提交。
5. 切到“成员”查看每位成员的提交概况和群组邀请码。

## 信息架构

- 群组
  - 群组名称
  - 群组副标题
  - 邀请码
  - 成员列表
  - 默认可见性
- 成员
  - 昵称
  - 头像符号
  - 角色
  - 加入时间
- 持仓
  - 标的代码
  - 标的名称
  - 市场
  - 数量
  - 平均成本
  - 现价
  - 币种
  - 可见性
  - 备注
  - 更新时间

## 可见性设计

- 完整可见：成员可看到数量、成本、市值和浮盈亏。
- 隐藏成本：成员可看到标的和市值，不展示成本。
- 仅标的：成员只看到用户持有哪些标的。

生产版本建议增加群组级规则，例如“必须自己提交后才能查看别人”“仅管理员可见明细”“群组只展示聚合不展示成员明细”。

## 风险和合规边界

- 不提供买卖建议，不做收益承诺。
- 不展示“排行榜式收益率刺激”作为核心体验，避免把产品推向投机竞赛。
- 持仓属于敏感金融数据，生产版本必须有清晰的权限、删除、导出和审计能力。
- 不同币种不能直接相加，首版已经按币种分别汇总。

## 下一版建议

- 登录：Apple 登录、手机号或邮箱。
- 群组邀请：邀请码加入、邀请链接、管理员审批。
- 数据持久化：Supabase、Firebase 或自建后端。
- 实时同步：群组成员提交后其他人自动刷新。
- 行情接入：按市场接入行情源，并标明行情延迟。
- 评论和反应：围绕单笔持仓讨论理由、风险和计划。
- 导入：支持 CSV、券商截图 OCR 或手工批量录入。
- 通知：成员新增仓位、仓位大幅变化、共识标的变化。
